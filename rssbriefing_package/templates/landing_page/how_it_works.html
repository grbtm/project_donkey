So, how does this actually work? 🤔


1. Humans: Collect and curate trending topics 🧐

We collect the trending headlines of the past 24h from a range of aggregators. Afterwards we handpick a small number of topics which we define as the set of current trending news. For that we focus on the most important global headlines.

2. AI: Select the most relevant feed posts from the past 24h 🔍

Based on the list of rss and Atom feeds of an user, a machine learning model picks the feed posts which are most similar to the above defined set of trending news.

3. AI: Summarize the picked articles 📝

Afterwards we collect the full articles of the selected feed posts and use another machine learning model to summarize the articles. In the end you get a summary for each trending topic, based on your list of news sources.

4. Provide the briefing to each user 🗞

Now that the custom briefing is generated, you can log in to your rssBriefing account and view your personal briefing - based on the current trending news, summarized according to your list of feeds.



Technologies we use:

gensim
We trained a document similarity model based on gensim's implementation of Doc2Vec and we use gensim's built-in summarization model.
Flask
The app is written using the Python micro web framework Flask.
Feedparser
To parse rss and Atom feeds, as the name suggests...
newspaper3k
The newspaper3k library allows us to parse the actual news articles.
AWS Elastic Beanstalk
For hosting.
